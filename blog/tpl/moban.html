Juicer

随着移动端的渐渐兴起,"流量"、"性能"，"速度",成为关键字,模板引擎以速度快,身板小,易上手,取代传统框架渲染数据 愈演愈烈. 各种方法雨后春笋,络绎不绝,但大都殊途同归,大同小异,这里以Juicer为例

核心模块:

HTML:

< div id="list">< /div>

< script id="tpl" type="text/template">

< ul>

{@each list as it,index}

< li>${it.name} (index: ${index})< /li>

{@/each}

{@each blah as it}

< li>

num: ${it.num} < br/>

{@if it.num==3}

{@each it.inner as it2}

${it2.time} < br/>

{@/each}

{@/if}

< /li>

{@/each}

< /ul>

< /script>

Javascript 代码:

var data = {

list : [

{name:'aaaa',show:true}

{name:'bbbb',show:false}

{name:'aaaa',show:true}

],

blash : [

{num:1}

{num:2}

{num:3,inner:[

{'time':'15:00'}

{'time':'16:00'}

{'time':'17:00'}

{'time':'18:00'}

]},

{num:4}

]

};

var tpl = document.getElementById('tpl').innerHTML;

var html = juicer(tpl,data);

list.innerHTML = data;

语法

${变量}

${name}

${name|function}

${name|function,arg1,arg2}

内联辅助函数 {@helper} … {@/helper}

循环遍历{@each} … {@/each}

{@each list as item, index}

${item.prop}

${index} // 当前索引

{@/each}

判断 {@if} … {@else if} … {@else} … {@/if}

{@each list as item, index}

{@if index===3}

the index is 3, the value is ${item.prop}

{@else if index===4}

the index is 3, the value is ${item.prop}

{@else}

the index is 3, the value is ${item.prop}

{@/if}

{@/each}

注释

{# 这里是注释内容}

辅助循环

{@each i in range(5, 10)}

${i} // 输出5;6;7;8;9;

{@/each}
